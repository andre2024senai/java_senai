Encapsulamento

Proteger e controlar o acesso aos dados

🎯 O que é?

Encapsulamento é esconder os detalhes internos de uma classe e expor apenas o necessário através de métodos públicos (getters e setters).

🌍 Analogia do Mundo Real:
Imagine um carro: você usa o volante, pedais e câmbio (interface pública), mas NÃO mexe diretamente no motor (detalhes internos protegidos).

💻 Código Exemplo

SEM Encapsulamento

public class Conta { public double saldo; } // No Main: Conta c = new Conta(); c.saldo = -1000; // ❌ Aceita valor inválido!

COM Encapsulamento

public class Conta { private double saldo; public void depositar(double valor) { if (valor > 0) { saldo += valor; } } }

🔑 Palavras-chave

private
Só a própria classe acessa
→
protected
Classe + Subclasses
→
public
Todos acessam
💡 Regra de Ouro: Atributos sempre private, métodos public quando necessário!
👨‍👦

Herança

Reutilizar código através de hierarquias

🎯 O que é?

Herança permite que uma classe herde atributos e métodos de outra classe, promovendo reutilização de código.

🌍 Analogia do Mundo Real:
Pessoa (pai) tem nome e idade
Aluno (filho) HERDA nome e idade + adiciona matrícula

📊 Diagrama Visual

Pessoa
- nome
- idade
+ exibirDados()
↓
extends (herda)
Aluno
↑ HERDA: nome, idade, exibirDados()
+ matricula (novo)
+ estudar() (novo)

💻 Código Exemplo

// Superclasse (Pai) public class Pessoa { private String nome; private int idade; public Pessoa(String nome, int idade) { this.nome = nome; this.idade = idade; } public void exibirDados() { System.out.println("Nome: " + nome); } } // Subclasse (Filho) public class Aluno extends Pessoa { // ← HERANÇA! private String matricula; public Aluno(String nome, int idade, String mat) { super(nome, idade); // ← Chama construtor do pai this.matricula = mat; } public void estudar() { System.out.println("Estudando..."); } }
💡 Teste IS-A: "Aluno É UMA Pessoa?" → SIM! ✓ Então pode herdar!

Polimorfismo

Muitas formas para o mesmo método

🎯 O que é?

Polimorfismo significa "muitas formas". Um mesmo método pode ter comportamentos diferentes dependendo do objeto.

🌍 Analogia do Mundo Real:
Comando "emitir som":
🐕 Cachorro → "Au au!"
🐱 Gato → "Miau!"
🐮 Vaca → "Muuu!"
Mesmo comando, comportamentos diferentes!

💻 Código Exemplo

// Classe pai public class Animal { public void fazerSom() { System.out.println("Som genérico"); } } // Subclasses com comportamentos diferentes public class Cachorro extends Animal { @Override public void fazerSom() { System.out.println("Au au!"); } } public class Gato extends Animal { @Override public void fazerSom() { System.out.println("Miau!"); } } // No Main - POLIMORFISMO em ação! Animal a1 = new Cachorro(); Animal a2 = new Gato(); a1.fazerSom(); // "Au au!" a2.fazerSom(); // "Miau!"

🔄 Tipos de Polimorfismo

Sobrescrita
@Override
Mesmo método, comportamento diferente
Sobrecarga
Mesmo nome, parâmetros diferentes
💡 Chave do Polimorfismo: Usar @Override para mudar o comportamento herdado!
🎨

Abstração

Definir contratos sem implementação

🎯 O que é?

Abstração é esconder a complexidade e mostrar apenas o essencial. Define "o que fazer" sem especificar "como fazer".

🌍 Analogia do Mundo Real:
Controle remoto da TV: Você aperta "ligar" sem saber os detalhes eletrônicos internos. A abstração esconde a complexidade!

💻 Código Exemplo - Classe Abstrata

// Classe ABSTRATA - não pode ser instanciada public abstract class Forma { private String cor; // Método ABSTRATO - sem implementação public abstract double calcularArea(); // Método CONCRETO - com implementação public void exibir() { System.out.println("Cor: " + cor); } } // Subclasse DEVE implementar métodos abstratos public class Circulo extends Forma { private double raio; @Override public double calcularArea() { return Math.PI * raio * raio; } }

💻 Código Exemplo - Interface

// INTERFACE - 100% abstrato (antes do Java 8) public interface Pagavel { void processarPagamento(double valor); boolean validarPagamento(); } // Classe IMPLEMENTA interface public class CartaoCredito implements Pagavel { @Override public void processarPagamento(double valor) { System.out.println("Pagando R$ " + valor); } @Override public boolean validarPagamento() { return true; } }

⚖️ Classe Abstrata vs Interface

Classe Abstrata

  • Pode ter métodos concretos
  • Pode ter atributos
  • Usa extends
  • Só herda de UMA classe

Interface

  • Só métodos abstratos*
  • Sem atributos (só constantes)
  • Usa implements
  • Pode implementar VÁRIAS
*Java 8+ permite default methods
💡 Quando usar: Use Interface para "capacidades" (Voavel, Nadavel) e Classe Abstrata para "tipos" (Animal, Veiculo).
